Chapter 01

Origin Story

// how a data-entry tool became a world language

In 1993, three engineers at PUC-Rio were tasked with building a scripting layer for Petrobras — Brazil's state oil giant. They had no idea they were creating something that would outlive every piece of software they'd ever write.

Roberto Ierusalimschy, Luiz Henrique de Figueiredo, and Waldemar Celes chose a bold philosophy: build a language so small and so embeddable that any application could host it. No standard file I/O. No networking. No threads. Just the pure kernel of a programming language — and let the host application decide everything else.

The name "Lua" means Moon in Portuguese. It was partly named as a counterpoint to "Sol" (Sun), a data-description language that had preceded it at PUC-Rio. The Moon doesn't shine on its own — it reflects light from something else. That metaphor perfectly describes Lua: it amplifies the host application, never trying to outshine it.

-- Variables: local vs global (always prefer local)
local x = 42          -- lexically scoped
y   = 99          -- global (lives in _G table)

-- Multiple assignment — swap without temp variable
local a, b = 1, 2
a, b = b, a        -- a=2, b=1  ✨

-- String operations
local s = "Lua"
print(#s)             -- 3  (length operator)
print(s:upper())      -- "LUA"  (method syntax)
print(s:rep(3, "-"))  -- "Lua-Lua-Lua"

-- The gotcha everyone hits: 0 is truthy!
if 0  then print("truthy")  end  -- prints!
if "" then print("truthy")  end  -- prints!
-- Only nil and false are falsy.

-- Multiple return values — no tuple wrapping needed
local function stats(t)
  local sum = 0
  for _, v in ipairs(t) do sum = sum + v end
  return sum, sum / #t    -- two return values
end

local total, avg = stats({10, 20, 30})
print(total, avg)   -- 60   20.0

-- Closures: functions capture their upvalues
local function adder(n)
  return function(x) return x + n end
end
local add5 = adder(5)
print(add5(10))       -- 15
🔥
// The local Keyword Is Not OptionalWithout local, every variable is global — it becomes a key in the _G table. In an embedded application, globals are visible to the C host, which is sometimes intentional. But inside tight loops, local variable access is 2–3x faster than global lookups because locals are stored in registers.
Chapter 03

Tables: The Everything Type

// one data structure to replace them all

What if you could design an entire programming language and give it only one data structure — and make that choice so brilliant that nobody ever needed anything else?

Lua's table is simultaneously an array, a hash map, a set, a stack, a queue, a namespace, a module, and an object. It's implemented as a hybrid: integer keys are stored in a contiguous C array for O(1) access and cache efficiency; all other keys live in a hash table. Lua automatically migrates keys between halves as the table grows.

lua — tables as everything 
-- As an array (1-indexed!)
local arr = { "mercury", "venus", "earth" }
print(arr[1])        -- "mercury"
arr[#arr+1] = "mars"  -- append

-- As a hash map
local http = { GET=200, POST=201, DELETE=204, NOT_FOUND=404 }
print(http.GET)       -- 200

-- As a set (keys = members, values = true)
local function Set(items)
  local s = {}
  for _, v in ipairs(items) do s[v] = true end
  return s
end
local fruits = Set({"apple", "mango", "banana"})
print(fruits["mango"])  -- true
print(fruits["grape"])  -- nil (not in set)

-- As a module (namespace)
local M = {}

function M.hello() return "hello from module" end
function M.bye()   return "goodbye" end

return M   -- standard module pattern

-- table.* standard library
local t = {3,1,4,1,5,9}
table.sort(t)
print(table.concat(t, ", "))  -- 1, 1, 3, 4, 5, 9
table.insert(t, 2, 99)         -- insert 99 at index 2
table.remove(t, 1)             -- remove first element
🧠
// Implementation Detail Worth KnowingThe # length operator on tables returns a border — an index where t[i] ~= nil and t[i+1] == nil. For sequences (arrays with no holes), this is the length. For tables with holes, the result is undefined. This trips up virtually every intermediate Lua programmer.
Chapter 04

Metatables & Magic

// operator overloading, inheritance, and the secret behind lua's OOP

Metatables are the most powerful — and most misunderstood — feature in Lua. They let you attach behaviour to tables: define what happens when you add two tables, compare them, access a missing key, or call a table like a function. The entire OOP system in Lua isn't built in — it's user-constructed from metatables.

MetamethodTriggered byCommon use case
__indext[k] — key missingInheritance chain, default values, lazy loading
__newindext[k] = v — new keyRead-only tables, change tracking, validation
__add / __suba + b, a - bVector / matrix arithmetic operators
__mul / __diva * b, a / bScalar multiplication, quaternion math
__eq / __lt / __le==, <, <=Value equality, sorted data structures
__callt(...)Callable objects, functor pattern
__tostringtostring(t)Debug output, serialisation
__len#tCustom length for non-sequence tables
__concata .. bString-like objects
__gcGC collectionResource cleanup, file handles, sockets
lua — building a class system with metatables 
-- A fully functional class with inheritance
local Animal = {}
Animal.__index = Animal

function Animal.new(name, sound)
  return setmetatable({ name=name, sound=sound }, Animal)
end

function Animal:speak()
  print(self.name .. " says: " .. self.sound)
end

function Animal:__tostring()
  return "Animal(" .. self.name .. ")"
end

-- Subclass: Dog inherits from Animal
local Dog = setmetatable({}, { __index = Animal })
Dog.__index = Dog

function Dog.new(name)
  local self = Animal.new(name, "Woof")
  return setmetatable(self, Dog)
end

function Dog:fetch(item)
  print(self.name .. " fetches the " .. item)
end

local rex = Dog.new("Rex")
rex:speak()      -- Rex says: Woof  (inherited)
rex:fetch("ball") -- Rex fetches the ball
print(tostring(rex))  -- Animal(Rex)

-- Proxy table: intercept ALL reads and writes
local function readonly(t)
  return setmetatable({}, {
    __index    = t,
    __newindex = function() error("read-only!") end,
    __len      = function() return #t end,
  })
end
Chapter 05

Coroutines

// cooperative multitasking that arrived 15 years before python's async/await

Lua's coroutines aren't just a feature — they're proof that the language designers understood something fundamental about concurrency that the industry took two more decades to grasp.

A Lua coroutine is a thread of execution that yields voluntarily, transferring control back to the caller. There's no preemption, no OS involvement, no context switching overhead. The entire mechanism is cooperative: a coroutine runs until it calls coroutine.yield(), pauses itself, and waits to be resumed.

lua — coroutines: three killer patterns 
━━━ Pattern 1: Generator (infinite sequence) ━━━
local function range(from, to, step)
  step = step or 1
  return coroutine.wrap(function()
    for i = from, to, step do
      coroutine.yield(i)
    end
  end)
end

for v in range(1, 10, 2) do
  io.write(v .. " ")  -- 1 3 5 7 9
end

━━━ Pattern 2: Async I/O scheduler (mini event loop) ━━━
local tasks = {}

local function spawn(fn)
  tasks[#tasks+1] = coroutine.create(fn)
end

local function run()
  while #tasks > 0 do
    local co = table.remove(tasks, 1)
    local ok, err = coroutine.resume(co)
    if coroutine.status(co) == "suspended" then
      tasks[#tasks+1] = co  -- reschedule
    end
  end
end

spawn(function()
  print("task A: step 1"); coroutine.yield()
  print("task A: step 2")
end)
spawn(function()
  print("task B: step 1"); coroutine.yield()
  print("task B: step 2")
end)
run()
-- task A: step 1 / task B: step 1 / task A: step 2 / task B: step 2

━━━ Pattern 3: Bidirectional channel (value passing) ━━━
local co = coroutine.create(function(seed)
  local val = seed
  while true do
    val = coroutine.yield(val * 2)  -- yield sends, resume receives
  end
end)

coroutine.resume(co, 1)       -- start: seed=1
local _, v = coroutine.resume(co, 5)
print(v)  -- 10  (5*2)
🎮
// Real World: World of WarcraftEvery WoW addon that sequences combat actions, waits for animations, or runs AI behaviour trees uses Lua coroutines. The entire addon system runs inside the game's Lua VM. When your character casts a spell, waits for a cooldown, and then casts another — that's a coroutine sleeping and waking.

The C DNA

// why the c api is lua's greatest feature

Lua was designed from day one to be embedded. The C API is not a bolt-on — it's the foundation. Every interaction between a host application and Lua goes through a single consistent mechanism: the virtual stack. Push values onto it. Call Lua functions. Pop results off it. That's the whole API.

c — embedding lua in your application 
// Expose a fast C function to Lua scripts
static int l_fast_sqrt(lua_State *L) {
    double n = luaL_checknumber(L, 1);  // arg 1 from stack
    lua_pushnumber(L, sqrt(n));          // push result
    return 1;                              // 1 return value
}

// Register a whole library
static const luaL_Reg mylib[] = {
    { "sqrt",   l_fast_sqrt  },
    { "sin",    l_fast_sin   },
    { NULL,     NULL          }  // sentinel
};

int luaopen_mylib(lua_State *L) {
    luaL_newlib(L, mylib);
    return 1;
}

// Run a Lua script from C
lua_State *L = luaL_newstate();
luaL_openlibs(L);
luaL_requiref(L, "mylib", luaopen_mylib, 1);
luaL_dofile(L, "script.lua");
lua_close(L);

The design of the Lua-C API is a masterclass in the right level of abstraction. It's low enough to give you full control, high enough to spare you pointer arithmetic hell.

Chapter 07

LuaJIT: Speed Demon

// when a scripting language starts competing with c

Mike Pall's LuaJIT is arguably the most impressive piece of compiler engineering ever written by one person. It implements a tracing JIT compiler for Lua that, for numerical workloads, regularly matches the performance of compiled C — while remaining fully compatible with standard Lua 5.1.

// Performance Relative to CPython (higher = faster)
LuaJIT (JIT on)~50–100×
Lua 5.5 (interpreter)~5–10×
CPython 3.12baseline 1×
PyPy (tracing JIT)~4–8×
Node.js V8~10–20×
benchmarks vary by workload type — numerical/loop-heavy code shows largest gains

LuaJIT's FFI (Foreign Function Interface) is particularly groundbreaking: it lets you call C functions and access C data structures from Lua with zero overhead — no wrapper code, no bindings — just declare the C prototype as a string and call it directly.

luajit ffi — call c with zero overhead 
local ffi = require("ffi")

-- Declare C prototypes directly as strings
ffi.cdef[[
  double sqrt(double x);
  int    printf(const char *fmt, ...);
  void  *malloc(size_t size);
  void   free(void *ptr);
]]

-- Call them directly — zero marshalling overhead
print(ffi.C.sqrt(2))        -- 1.4142135623731
ffi.C.printf("Hello %s\n", "LuaJIT")

-- Define C structs and work with them
ffi.cdef[[
  typedef struct { float x, y, z; } Vec3;
]]

local v = ffi.new("Vec3", 1.0, 2.0, 3.0)
print(v.x, v.y, v.z)  -- 1.0  2.0  3.0

Chapter 10

Where the Moon is Headed

// lua in 2026 and beyond

Lua is not a language chasing trends. No corporate foundation. No conference circuit. No social media presence. Just three professors at PUC-Rio — Roberto Ierusalimschy, Luiz Henrique de Figueiredo, and Waldemar Celes — releasing software only when it's right. Lua 5.5 dropped on December 22, 2025, five years after 5.4. That cadence is the feature.

Lua 5.5 is the most impactful release since 5.3. Large arrays now use ~60% less memory — transformational for embedded systems running on 256KB RAM. The generational GC's major collections are now fully incremental, eliminating the stop-the-world pauses that plagued Redis scripts and Nginx request handlers in 5.4. The new global declaration keyword finally lets programmers catch typo-induced global leaks at compile time, and for-loop control variables are now immutable by default — a safety win that aligns Lua with modern language design.

Luau from Roblox hit a landmark milestone in November 2025 when its ground-up rewritten type solver graduated from Studio Beta to general release. It now powers Alan Wake 2, Warframe, Second Life (via SLua), and Roblox's 3M+ developers. The spec gap between Luau and standard Lua is widening in the best possible way — Luau is Lua's TypeScript moment, and other games are taking notice.

The Neovim ecosystem, anchored by package managers like lazy.nvim and thousands of Lua-native plugins, has become one of the most active Lua communities worldwide. In February 2026, Lua became an SPI (Software in the Public Interest) associated project — a governance milestone ensuring long-term stewardship independent of any single institution. Wikipedia uses Lua as its template language. The Moon is far from setting.

lua 5.5 — what's new in the latest release 
━━━ 1. Global Declarations — catch typos at compile time ━━━
-- Lua 5.4: typo silently creates a new global
resonse = get("url")      -- 'response' misspelled → silent bug

-- Lua 5.5: declare globals explicitly
global response            -- declare intent
response = get("url")     -- 'resonse' would now error ✓

━━━ 2. Read-only for-loop variables ━━━
for i = 1, 10 do
  -- i = i + 1   ← compile error in 5.5! i is now const
  print(i)
end

━━━ 3. table.create() — pre-allocated arrays ━━━
-- Pre-allocate 10,000 slots → no rehashing, ~60% less RAM
local arr = table.create(10000)
for i = 1, 10000 do
  arr[i] = i * i
end
-- In Lua 5.4, large arrays wasted >40% RAM in alignment padding
-- 5.5 compact arrays store sequential integer keys tightly

━━━ 4. Incremental major GC — no stop-the-world pauses ━━━
-- Lua 5.4: major GC sweeps entire heap atomically (spike!)
-- Lua 5.5: major GC runs incrementally in small steps
-- Result: Redis scripts get predictable latency,
--         Nginx never pauses, games never drop frames
collectgarbage("incremental")   -- default in 5.5

━━━ 5. utf8.offset returns final character position ━━━
local s = "Héllo"
print(utf8.offset(s, 2))   -- byte pos of 2nd character
print(utf8.offset(s, -1))  -- final char pos (new in 5.5)
🔭
// Lua 5.5 — Current Stable (Released Dec 22, 2025) The biggest release in five years. Key features: explicit global declarations (catch typo-globals at compile time), 60% memory reduction for large arrays via compact storage, incremental major GC (no more stop-the-world pauses), read-only for-loop variables, table.create(), named vararg tables, external strings, luaL_openselectedlibs, and dynamic readline loading. LuaRocks and Teal are mid-migration to full 5.5 support.

The philosophy that made Lua great in 1993 — radical minimalism, embeddability, one powerful data structure — keeps it indispensable in 2026. Five years of patience delivered Lua 5.5's 60% memory savings and pause-free GC. That's not a language stagnating. That's a language being done right.

The Moon doesn't shout. It doesn't trend. It doesn't have a build system or a package manifest. It just quietly illuminates — and has done so for thirty-two years.
